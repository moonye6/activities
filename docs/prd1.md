# 活动组织小程序 PRD v1.1

## 一、产品定位

面向熟人社交的轻量活动组织工具。  
核心价值：**极简发起 → 精准找人 → 顺利成局**，解决"约人麻烦、沟通成本高、活动黄了"的痛点。

---

## 二、功能优先级

### 🔴 核心功能（MVP，必须优先实现）

| 功能 | 说明 |
|------|------|
| 一键发布活动 | 只填必要字段：时间、活动类型，其余可选 |
| 活动可见性控制 | 朋友可见 / 标签可见 / 公开 |
| 时间匹配参与者 | 系统根据时间可用性推荐合适人选 |
| 参与确认与状态统计 | 参加 / 拒绝 / 暂定，实时统计人数 |
| 简单提醒 | 活动开始前推送或小程序内提醒 |

> 💡 核心理由：解决用户最痛的点——快速约到人，低沟通成本，活动顺利执行。

---

### 🟡 高级功能（可迭代实现）

| 功能 | 说明 |
|------|------|
| 自定义活动模板 | 重复活动快速创建（吃饭、运动、聚会等） |
| 标签管理 | 系统标签（公司/部门/校友）+ 个人自定义兴趣圈 |
| 智能推荐参与者 | 根据标签、活跃度、历史参与记录推荐 |
| 地点推荐 | 餐厅、场馆等，根据人数、评分、位置推荐 |
| 多人/定向邀请 | 单人定向约 或 多人开放招募 |
| 活动后反馈 | 照片分享、心得、参与者互评（信任机制） |

> 💡 高级理由：提升效率和社交体验，但不影响 MVP 能否验证核心价值。

---

### ⚪ 潜在伪需求（先不做，待验证）

| 功能 | 风险原因 |
|------|----------|
| 匿名发布 / 模糊位置 | 熟人社交场景中需求低 |
| 复杂标签体系（多层嵌套、兴趣权重） | 用户学习成本高，使用频率低 |
| 活动数据分析（参与率、活跃度统计） | 早期用户量小，数据价值有限 |
| 付费增值（高级匹配、付费曝光等） | 未验证核心留存前过早商业化风险大 |
| 与其他社交平台深度整合 | 开发成本高，收益不确定 |

> 💡 风险原因：用户可能不关心或使用频率低，先投入成本容易浪费资源。

## 四、核心功能详细说明

### 4.1 活动创建

- **一键发布**：只需填写时间和活动类型，系统自动匹配潜在参与者
- **活动信息设置**：
  - 必填：时间、活动类型
  - 选填：地点、活动标题、人数上限
- **可见范围**：朋友可见 / 特定标签可见 / 公开
- **标签选择**：系统标签（公司、部门、校友等）+ 个人标签（自定义兴趣圈）
- **人数限制**：单人定向约 / 多人开放招募

### 4.2 私密性与可见性控制

- **朋友可见**：仅互相关注的好友可见
- **标签可见**：仅指定标签成员可见（如"同部门"）
- **公开**：所有人均可见

### 4.3 参与确认与状态

- 参与者操作：**参加 / 拒绝 / 暂定**
- 系统实时统计确认人数，自动更新活动状态
- 活动状态流转：招募中 → 已确认 → 已结束 / 已取消

### 4.4 提醒

- 活动开始前推送通知（小程序订阅消息或站内消息）

## 五、MVP 用户流程

```
用户进入小程序
        │
        ▼
【创建活动】
 ├─ 填写：时间、活动类型（必填）
 ├─ 选填：地点、标题、人数
 ├─ 设置：可见范围、标签
        │
        ▼
【参与者发现 & 确认】
 ├─ 系统根据可见范围 + 标签推送给潜在参与者
 ├─ 参与者操作：参加 / 拒绝 / 暂定
 └─ 发起人实时查看参与人数与状态
        │
        ▼
【活动提醒】
 └─ 活动前自动提醒所有已确认参与者
        │
        ▼
【活动完成】
 └─（后续迭代）照片分享、互评
```

```json
{
  "_id": "act_001",
  "title": "周五一起吃火锅",
  "type": "dinner",
  "time": "2026-04-10 19:00",
  "location": "三里屯",

  "maxCount": 6,

  "status": "recruiting",
  "rsvps": [
    { "userId": "user_002", "status": "yes" },
    { "userId": "user_003", "status": "maybe" }
  ],
  "createdAt": 1744000000000

```

**活动类型（type）**：`dinner` / `sport` / `party` / `movie` / `course` / `other`

**可见范围（visibility）**：`friends` / `public` / `tags`

**活动状态（status）**：`recruiting` / `confirmed` / `ended` / `cancelled`

**RSVP 状态**：`yes` / `maybe` / `no`

## 七、核心价值总结

| 价值点 | 说明 |
|--------|------|
| 极简创建 | 最少填写字段，降低发起门槛 |
| 标签与私密性 | 精准触达合适的人，保护隐私 |
| 实时统计 | 发起人随时掌握参与情况 |
| 可扩展 | 迭代路线清晰，不过度设计 |
